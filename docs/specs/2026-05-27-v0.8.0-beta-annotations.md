# v0.8.0-beta — Annotation Queues Design Spec

> **Status**: Draft
> **Target Release**: v0.8.0-beta
> **Depends on**: v0.7.0-beta (Evaluations)
> **Frozen At**: —

---

## 1. Goal

Add a **team-based manual annotation workflow** that lets project members collaboratively review traces, assign scores, and approve annotations. Annotation queues bridge the gap between automated evals (v0.7.0) and ground-truth quality signals.

**Exit**: Users can create annotation queues, populate them with traces, assign items to team members, annotate via a dedicated workbench UI, and track completion progress.

---

## 2. Philosophy

Constell annotations are **lightweight and queue-driven**:

1. **Queue-centric** — Annotations happen inside queues, not as ad-hoc trace comments
2. **Score-compatible** — Annotation results are just scores (`source=ANNOTATION`), reusing all existing score infrastructure
3. **Optional two-tier review** — Simple queues have single annotator; review queues have annotator + reviewer

**Not in scope:**

- Real-time collaborative annotation (locking/conflict resolution)
- Inter-annotator agreement (IAA) metrics
- ML-assisted pre-labeling
- Public API for annotation queues

## 4. Data Model

### 4.1 PostgreSQL (Prisma)

**`AnnotationQueue`** — A named queue of traces to annotate.

```prisma
model AnnotationQueue {
  id            String   @id @default(cuid())
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @default(now()) @updatedAt @map("updated_at")
  projectId     String   @map("project_id")
  project       Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  name          String
  description   String?
  scoreConfigId String?  @map("score_config_id")
  scoreConfig   ScoreConfig? @relation(fields: [scoreConfigId], references: [id], onDelete: SetNull)

  items         AnnotationQueueItem[]

  @@unique([projectId, name])
  @@index([projectId])
  @@map("annotation_queues")
}
```

**`AnnotationQueueItem`** — A single trace assigned to the queue.

```prisma
enum AnnotationStatus {
  PENDING
  ASSIGNED
  ANNOTATED
  REVIEWED
}

model AnnotationQueueItem {
  id            String   @id @default(cuid())
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @default(now()) @updatedAt @map("updated_at")
  queueId       String   @map("queue_id")
  queue         AnnotationQueue @relation(fields: [queueId], references: [id], onDelete: Cascade)
  traceId       String   @map("trace_id")

  status        AnnotationStatus @default(PENDING)
  assignedTo    String?  @map("assigned_to")
  annotator     User?    @relation(fields: [assignedTo], references: [id], onDelete: SetNull)

  scoreValue    Float?   @map("score_value")
  stringValue   String?  @map("string_value")
  comment       String?

  reviewedBy    String?  @map("reviewed_by")
  reviewer      User?    @relation(fields: [reviewedBy], references: [id], onDelete: SetNull)

  @@index([queueId, status])
  @@index([assignedTo, status])
  @@map("annotation_queue_items")
}
```

Add to `Project`:

```prisma
  annotationQueues AnnotationQueue[]
```

Add to `User`:

```prisma
  assignedItems   AnnotationQueueItem[] @relation("Annotator")
  reviewedItems   AnnotationQueueItem[] @relation("Reviewer")
```

> **Rationale**: Items are independent of scores until annotation. When an item is annotated, a `Score` record is created with `source=ANNOTATION`. The item stores the score locally for quick queue status queries.

## 5. Worker

No worker needed. Annotation is entirely synchronous via tRPC.

## 6. API Design

### 6.1 tRPC `annotations` Router

```typescript
annotations.queueList
  Input: { projectId?: string }
  Output: Array<{ id, name, description, scoreConfigName, itemCount, completedCount }>

annotations.queueCreate
  Input: { projectId?: string, name: string, description?: string, scoreConfigId?: string }
  Output: { id: string }

annotations.queueDelete
  Input: { projectId?: string, id: string }
  Output: { success: boolean }

annotations.queuePopulate
  Input: { projectId?: string, queueId: string, traceIds: string[] }
  Output: { added: number }

annotations.itemList
  Input: {
    projectId?: string;
    queueId: string;
    status?: AnnotationStatus;
    assignedTo?: string;
    limit?: number;
    offset?: number;
  }
  Output: { items: Array<{ id, traceId, status, assignedTo, scoreValue, comment }>, total: number }

annotations.itemAssign
  Input: { projectId?: string, itemIds: string[], userId?: string }
  // userId null = unassign
  Output: { updated: number }

annotations.itemAnnotate
  Input: {
    projectId?: string;
    itemId: string;
    scoreValue?: number;
    stringValue?: string;
    comment?: string;
  }
  // Creates Score record with source=ANNOTATION
  // Updates item status to ANNOTATED
  Output: { id: string }

annotations.itemReview
  Input: { projectId?: string, itemId: string, approved: boolean }
  // approved=true → status=REVIEWED, Score kept
  // approved=false → status=ASSIGNED, Score deleted, scoreValue cleared
  Output: { id: string }

annotations.myAssignments
  Input: { projectId?: string, status?: AnnotationStatus }
  Output: Array<{ itemId, queueName, traceId, status }>

annotations.queueStats
  Input: { projectId?: string, queueId: string }
  Output: {
    total: number;
    pending: number;
    assigned: number;
    annotated: number;
    reviewed: number;
    averageScore: number | null;
  }
```

## 7. UI Design

### 7.1 Queue Management (`/annotations/queues`)

```
┌────────────────────────────────────────────────────────────┐
│  Annotation Queues                          [+ New Queue]  │
├────────────────────────────────────────────────────────────┤
│  Name              │ Config      │ Items │ Done │ Actions  │
│  ─────────────────────────────────────────────────────────│
│  Q2 Review         │ helpfulness │ 42    │ 38   │ Manage ▾ │
│  Safety Audit      │ safety_flag │ 100   │ 12   │ Manage ▾ │
└────────────────────────────────────────────────────────────┘
```

### 7.2 Queue Detail (`/annotations/queues/[id]`)

- Stats bar: total / pending / assigned / annotated / reviewed
- Item table with filters (status, assigned to)
- Bulk assign action
- "Populate from traces" button (opens trace picker)

### 7.3 Annotator Workbench (`/annotations/work`)

```
┌────────────────────────────────────────────────────────────┐
│  My Assignments — 5 remaining                              │
├────────────────────────────────────┬───────────────────────┤
│  Trace Detail                      │  Annotation Panel     │
│  ───────────────────────────────── │  ──────────────────── │
│  Input: {trace.input}              │  Score: [____] / 5    │
│                                    │                       │
│  Output: {trace.output}            │  Comment:             │
│                                    │  [                  ] │
│                                    │                       │
│                                    │  [Submit] [Skip]      │
└────────────────────────────────────┴───────────────────────┘
```

- Shows one item at a time
- Left: trace detail (input/output fetched via tRPC)
- Right: score input based on linked ScoreConfig type (numeric slider, boolean toggle, categorical dropdown)
- Submit → creates Score, marks item ANNOTATED, loads next item
- Skip → unassigns item, loads next

### 7.4 Review Panel (`/annotations/queues/[id]/review`)

- Lists ANNOTATED items awaiting review
- Shows annotator name, score, comment
- Approve → REVIEWED
- Reject → back to ASSIGNED, score cleared

## 8. Security

- All endpoints use `authedProcedure`
- `itemAssign` requires ADMIN or OWNER role (RBAC check via membership)
- `itemAnnotate` only allowed if `assignedTo === ctx.session.user.id`
- `itemReview` requires ADMIN or OWNER role
- Cross-project access returns 404

## 10. Post-v0.8.0 Ideas

- **IAA metrics** — Inter-annotator agreement (Cohen's kappa) for multi-annotator queues
- **Pre-labeling** — Populate queue with eval/ML-generated scores for annotators to confirm/edit
- **Public API** — Programmatic queue creation and item retrieval
- **Time-based assignment** — Auto-assign items round-robin to available annotators
