# v0.9.0-beta — Datasets & Experiments Spec

**Version:** 0.9.0-beta  
**Date:** 2026-05-28  
**Status:** Draft → Approved  
**Scope:** Dataset management, Run execution (active LLM invocation), Run comparison

---

## 1. Overview

### 1.1 Purpose

v0.9.0-beta introduces **Datasets & Experiments** — the connective tissue between Traces, Scores, Evals, and Annotations. A Dataset is a curated collection of test inputs (and optionally expected outputs). A Dataset Run actively executes a Prompt + Model configuration against all Dataset Items, producing Traces that can be scored by Eval Templates and compared side-by-side.

### 1.2 Design Goals

| Goal                              | Description                                                                                                                     |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **Reproducible experiments**      | Every Run snapshots its execution parameters in `metadata`, ensuring the experiment is replayable even if Presets change later. |
| **Flexible comparison**           | Compare any two or more Runs across arbitrary dimensions (model, prompt, temperature).                                          |
| **Minimal new infrastructure**    | Reuse existing `evalProcessor` LLM client, `eval-queue`, `ingestion-queue`, and `Score` infrastructure.                         |
| **Production-data feedback loop** | Convert live Traces/Observations into Dataset Items for regression testing.                                                     |

### 1.3 Non-Goals

- Dataset versioning (Langfuse-style `validFrom`/`validTo`) — keep `DatasetItem` immutable, update = delete + recreate.
- Automatic `expected_output` vs `actual_output` string-diff scoring — rely entirely on Eval Templates and existing Score system.
- Cross-Dataset Run comparison — Runs are scoped to a single Dataset.
- Real-time streaming during Run execution — batch mode only for v0.9.0.

## 2. Architecture

### 2.1 High-Level Flow

```
User creates DatasetRun
    │
    ▼
┌─────────────────┐
│  web (tRPC)     │ ──► 1. Auth check
│  datasetsRun    │ ──► 2. Resolve config (Preset or manual)
│  .create        │ ──► 3. Snapshot params into DatasetRun.metadata
│                 │ ──► 4. Enqueue to dataset-run-queue
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  worker         │ ──► 5. Dequeue DatasetRunJob
│ experiment      │ ──► 6. Fetch Dataset + Items
│ Processor       │ ──► 7. For each Item:
│                 │       a. Render prompt variables
│                 │       b. LLM call (shared LLM client)
│                 │       c. Ingest trace/observation
│                 │       d. Create DatasetRunItem
│                 │       e. Enqueue eval jobs
│                 │ ──► 8. Update Run status COMPLETED/FAILED
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  eval-queue     │ ──► 9. evalProcessor scores each trace
│  (existing)     │ ──► 10. Scores written to PG + CH
└─────────────────┘
    │
    ▼
User opens Run Comparison
    │
    ▼
┌─────────────────┐
│  web (tRPC)     │ ──► 11. Query DatasetItems × Runs
│  datasetsRun    │ ──► 12. Join scores via traceId
│  .compare       │ ──► 13. Return matrix view
└─────────────────┘
```

### 2.2 Key Design Decisions

| Decision               | Choice                                     | Rationale                                                         |
| ---------------------- | ------------------------------------------ | ----------------------------------------------------------------- |
| **Trace creation**     | Via ingestion pipeline (B)                 | Reuses cost calculator, latency enrichment, consistent data path. |
| **LLM client**         | Extract shared client from `evalProcessor` | Single source of truth for OpenAI-compatible API calls.           |
| **Run params storage** | Snapshot in `DatasetRun.metadata` JSON     | Reproducibility: Run result is independent of later Preset edits. |
| **Score source**       | Eval Templates only                        | No bespoke diff scoring; Eval system handles all metrics.         |
| **Concurrency**        | One worker job per `datasetRunId`          | BullMQ jobId = `datasetRunId` guarantees idempotency.             |

## 3. Database Schema

### 3.1 Prisma Models

```prisma
// ─── Datasets ───
model Dataset {
  id                   String         @id @default(cuid())
  projectId            String         @map("project_id")
  project              Project        @relation(fields: [projectId], references: [id], onDelete: Cascade)

  name                 String
  description          String?
  inputSchema          Json?          @map("input_schema")
  expectedOutputSchema Json?          @map("expected_output_schema")
  metadata             Json?

  items                DatasetItem[]
  runs                 DatasetRun[]

  createdAt            DateTime       @default(now()) @map("created_at")
  updatedAt            DateTime       @default(now()) @updatedAt @map("updated_at")

  @@unique([projectId, name])
  @@index([projectId])
  @@map("datasets")
}

enum DatasetItemStatus {
  ACTIVE
  ARCHIVED
}

model DatasetItem {
  id                  String            @id @default(cuid())
  projectId           String            @map("project_id")

  status              DatasetItemStatus @default(ACTIVE)
  input               Json?
  expectedOutput      Json?             @map("expected_output")
  metadata            Json?

  sourceTraceId       String?           @map("source_trace_id")
  sourceObservationId String?           @map("source_observation_id")

  datasetId           String            @map("dataset_id")
  dataset             Dataset           @relation(fields: [datasetId, projectId], references: [id, projectId], onDelete: Cascade)

  runItems            DatasetRunItem[]

  createdAt           DateTime          @default(now()) @map("created_at")
  updatedAt           DateTime          @default(now()) @updatedAt @map("updated_at")

  @@index([datasetId])
  @@index([sourceTraceId])
  @@index([sourceObservationId])
  @@map("dataset_items")
}

// ─── Run Presets ───
model DatasetRunPreset {
  id              String   @id @default(cuid())
  projectId       String   @map("project_id")
  project         Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)

  name            String
  description     String?
  promptId        String?  @map("prompt_id")
  promptVersion   Int?     @map("prompt_version")

  model           String
  modelParams     Json?    @map("model_params")
  evalTemplateIds String[] @map("eval_template_ids")

  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @default(now()) @updatedAt @map("updated_at")

  @@unique([projectId, name])
  @@map("dataset_run_presets")
}

// ─── Dataset Runs ───
enum DatasetRunStatus {
  PENDING
  RUNNING
  COMPLETED
  FAILED
  CANCELLED
}

model DatasetRun {
  id              String           @id @default(cuid())
  projectId       String           @map("project_id")

  name            String
  description     String?
  status          DatasetRunStatus @default(PENDING)
  metadata        Json?            // snapshot: { presetId?, promptId, model, modelParams, evalTemplateIds }

  datasetId       String           @map("dataset_id")
  dataset         Dataset          @relation(fields: [datasetId, projectId], references: [id, projectId], onDelete: Cascade)

  runItems        DatasetRunItem[]

  createdAt       DateTime         @default(now()) @map("created_at")
  updatedAt       DateTime         @default(now()) @updatedAt @map("updated_at")

  @@unique([datasetId, projectId, name])
  @@index([datasetId])
  @@index([status])
  @@map("dataset_runs")
}

model DatasetRunItem {
  id            String     @id @default(cuid())
  projectId     String     @map("project_id")

  datasetRunId  String     @map("dataset_run_id")
  datasetRun    DatasetRun @relation(fields: [datasetRunId, projectId], references: [id, projectId], onDelete: Cascade)

  datasetItemId String     @map("dataset_item_id")
  traceId       String     @map("trace_id")
  observationId String?    @map("observation_id")

  createdAt     DateTime   @default(now()) @map("created_at")
  updatedAt     DateTime   @default(now()) @updatedAt @map("updated_at")

  @@index([datasetRunId])
  @@index([datasetItemId])
  @@index([traceId])
  @@map("dataset_run_items")
}
```

### 3.2 Schema Notes

- `Dataset` and `DatasetRun` use `projectId` relations (consistent with existing models), not composite PKs like Langfuse.
- `DatasetRunPreset` is standalone and reusable across Datasets within a Project.
- `DatasetRun.metadata` is the **source of truth** for execution parameters; Preset is only a convenience at creation time.
- No soft-delete versioning on `DatasetItem` — delete + recreate for updates.

## 4. Worker Pipeline

### 4.1 Queue Contract

```typescript

export interface DatasetRunJob {
  projectId: string;
  datasetId: string;
  datasetRunId: string;
  // Execution params (copied from DatasetRun.metadata)
  promptId?: string;
  promptVersion?: number;
  model: string;
```

### 4.2 experimentProcessor

```typescript
// worker/src/queues/experimentProcessor.ts

export async function processExperimentJob(job: Job<DatasetRunJob>) {
  const { projectId, datasetId, datasetRunId, ...execParams } = job.data;

  // 1. Update status → RUNNING
  await prisma.datasetRun.update({
    where: { id: datasetRunId },
    data: { status: "RUNNING" },
  });

  const dataset = await prisma.dataset.findUnique({
    where: { id: datasetId },
    include: { items: { where: { status: "ACTIVE" } } },
  });

  if (!dataset || dataset.items.length === 0) {
    await failRun(datasetRunId, "No active items in dataset");
    return;
  }

  const failures: Array<{ itemId: string; reason: string }> = [];

  // 2. Process each item
  for (const item of dataset.items) {
    try {
      // 2a. Resolve prompt content
      const prompt = execParams.promptId
        ? await resolvePrompt(execParams.promptId, execParams.promptVersion)
        : null;

      // 2b. Render variables
      const rendered = prompt
        ? renderPromptVariables(prompt.content, item.input)
        : JSON.stringify(item.input);

      // 2c. LLM call
      const llmResponse = await callLlm({
        model: execParams.model,
        modelParams: execParams.modelParams,
        messages: [{ role: "user", content: rendered }],
      });

      // 2d. Ingest trace/observation via ingestion pipeline
      const { traceId, observationId } = await ingestLlmResult({
        projectId,
        runId: datasetRunId,
        datasetItemId: item.id,
        input: rendered,
        output: llmResponse.content,
        model: execParams.model,
        usage: llmResponse.usage,
        latencyMs: llmResponse.latencyMs,
      });

      // 2e. Create DatasetRunItem
      await prisma.datasetRunItem.create({
        data: {
          projectId,
          datasetRunId,
          datasetItemId: item.id,
          traceId,
          observationId,
        },
      });

      // 2f. Enqueue eval jobs
      for (const templateId of execParams.evalTemplateIds) {
        await enqueueEvalJob({
          projectId,
          traceId,
          observationId,
          evalTemplateId: templateId,
        });
      }
    } catch (err) {
      failures.push({ itemId: item.id, reason: (err as Error).message });
      // Continue with remaining items
    }
  }

  // 3. Finalize status
  const allFailed = failures.length === dataset.items.length;
  await prisma.datasetRun.update({
    where: { id: datasetRunId },
    data: {
      status: allFailed ? "FAILED" : "COMPLETED",
      metadata: {
        ...execParams,
        processedCount: dataset.items.length - failures.length,
        failureCount: failures.length,
        failures: failures.slice(0, 50), // cap stored failures
      },
    },
  });
}
```

### 4.3 Shared LLM Client Extraction

Extract OpenAI-compatible chat completion client from `evalProcessor` into a shared module:

```typescript
// packages/shared/src/server/llm/client.ts

export interface LlmCallParams {
  model: string;
  modelParams?: Record<string, unknown>;
  messages: Array<{ role: string; content: string }>;
}

export interface LlmCallResult {
  content: string;
  usage?: { prompt_tokens: number; completion_tokens: number; total_tokens: number };
  latencyMs: number;
}

export async function callLlm(params: LlmCallParams): Promise<LlmCallResult>;
```

### 4.4 Ingestion Integration

The processor constructs a standard ingestion event batch and enqueues it to `ingestion-queue`:

```typescript
async function ingestLlmResult(params: IngestParams) {
  const traceId = generateId();
  const observationId = generateId();

  const events = [
    buildTraceEvent({
      id: traceId,
      projectId: params.projectId,
      name: `dataset-run-${params.runId}`,
    }),
    buildObservationEvent({
      id: observationId,
      traceId,
      projectId: params.projectId,
      type: "GENERATION",
      input: params.input,
      output: params.output,
      model: params.model,
      usage: params.usage,
    }),
  ];

  await enqueueIngestionEvents(params.projectId, events);

  return { traceId, observationId };
}
```

### 4.5 Error Handling

| Error                   | Behavior                                             |
| ----------------------- | ---------------------------------------------------- |
| LLM API 429/5xx         | BullMQ retry with exponential backoff                |
| LLM API 400/401         | Non-retryable; skip item, record reason              |
| Missing prompt variable | Skip item, record "missing variable: X"              |
| Ingestion failure       | Retry via ingestion-queue; if still fails, skip item |
| All items skipped       | Run.status → FAILED                                  |
| Partial success         | Run.status → COMPLETED; metadata.failureCount > 0    |

### 6.2 Dataset Detail Page

**Items Tab:**

- Table: `# | Input (truncated JSON) | Expected Output | Source | Status`
- Actions: New Item (manual), Upload CSV, Add from Trace
- Source badge: "Manual" | "Trace: abc123"

**Runs Tab:**

- Table: `Name | Status (with progress) | Items | Avg Latency | Avg Cost | Actions`
- Status: PENDING → RUNNING (spinner + X/Y) → COMPLETED (green) / FAILED (red)
- Actions: View detail, Delete
- Bulk action: Checkbox selection → "Compare Selected" (navigates to `/compare?runs=...`)

### 6.4 Run Comparison Page (`/datasets/[id]/compare`)

**Layout:** Fixed first two columns (Item #, Input + Expected Output), variable Run columns to the right.

**Run Column per Item:**

```
┌─────────────────────┐
│ Output (truncated)  │
│ Score: 0.92 ✅      │
│ Lat: 1.2s | $0.04   │
```

**Score color coding:**

- Green dot: ≥ 0.8
- Yellow dot: 0.5 – 0.8
- Red dot: < 0.5

**Interactions:**

- Click cell → Drawer with full output + all scores + link to Trace detail
- "Change Selection" → reopen run picker (max 5)
